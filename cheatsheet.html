Quick reference for key ML concepts, metrics, and frameworks - everything you need at your fingertips

Data & Features Models Evaluation Serving Frameworks

Data & Feature Engineering

Text Features

  • Preprocessing: Lowercasing, tokenization, stopwords, stemming/lemma
  • Vectorization: BOW, TF-IDF (sparse), Word2Vec, BERT (dense)
  • Tokenization: BPE, WordPiece, SentencePiece
  • ID Encoding: Lookup table vs hashing (memory vs collision)

Image/Video Features

  • Preprocessing: Resize, normalize, decode frames
  • Embeddings: ResNet, CLIP, SimCLR, VideoMOCO
  • Augmentation: Crop, rotate, flip, color jitter
  • Video: Frame sampling, 3D convolutions, temporal pooling

Numerical Features

  • Scaling: StandardScaler, MinMax, RobustScaler
  • Bucketization: Equal-width, quantile-based, log transform
  • Missing values: Mean/median imputation, indicator column
  • Skewed data: Log transform, Box-Cox, Yeo-Johnson

Categorical Features

  • Low cardinality: One-hot encoding
  • High cardinality: Embedding layers, hashing
  • Feature crosses: Manual or DCN (Deep Cross Network)
  • Target encoding: Mean encoding with regularization

Model Architectures

Recommendation Systems

  • Content-based: Item features - similarity - recommend
  • Collaborative Filtering: User-item matrix factorization
  • Two-tower: User encoder + Item encoder, ANN retrieval
  • Sequential: Transformers over interaction history
  • MMOE: Multi-task with shared experts, task-specific towers

Ranking & Search

  • Pointwise: Score items independently (log loss)
  • Pairwise: Learn relative order (RankNet, LambdaRank)
  • Listwise: Optimize full ranking (LambdaMART, NDCG)
  • Retrieval - Ranking - Reranking: Multi-stage pipeline

Classification Models

  • Logistic Regression: Fast, interpretable, linear boundaries
  • XGBoost/LightGBM: Feature importance, no continual learning
  • DNN: Complex patterns, requires more data
  • Factorization Machines: Efficient feature crosses
  • DCN: Automatic cross features with DNN

Embeddings & Similarity

  • Contrastive Loss: Push similar together, dissimilar apart
  • Triplet Loss: Anchor, positive, negative
  • In-batch Negatives: Use other batch items as negatives
  • ANN Search: FAISS, ScaNN, HNSW, LSH

Evaluation Metrics

Classification

  • Precision: TP / (TP + FP) - quality of positives
  • Recall: TP / (TP + FN) - coverage of positives
  • F1: Harmonic mean of P & R
  • AUC-ROC: Ranking quality across thresholds
  • PR-AUC: Better for imbalanced data

Ranking

  • Precision@K: Relevant in top K
  • Recall@K: Coverage in top K
  • MRR: 1/rank of first relevant item
  • MAP: Mean AP across queries
  • NDCG: Graded relevance with position discount

Online Metrics

  • CTR: Clicks / Impressions
  • Conversion Rate: Purchases / Clicks
  • Dwell Time: Time spent on content
  • Session Time: Total engagement
  • Guardrails: Report rate, hide rate, churn

When to Use What

Binary relevance: Precision@K, Recall@K, MAP

Graded relevance: NDCG

Single relevant item: MRR

Imbalanced classes: PR-AUC over ROC-AUC

Calibration needed: Brier score, ECE

Diversity needed: Coverage, intra-list diversity

Serving & Production

Latency Optimization

  • Caching: Static features, precomputed embeddings
  • Batching: Combine requests for GPU efficiency
  • Distillation: Train smaller model from large teacher
  • Quantization: Reduce precision (FP16, INT8)
  • Pruning: Remove less important weights

Feature Store

  • Offline features: Batch computed, daily refresh
  • Online features: Real-time, streaming updates
  • Train-serve consistency: Same logic for both
  • Point-in-time correctness: Avoid data leakage

Deployment Strategy

  • Shadow mode: Run alongside, no user impact
  • Canary: Small % traffic, monitor metrics
  • A/B test: Statistical significance, holdout
  • Blue-green: Quick rollback capability

Monitoring

  • Data drift: PSI, KS test on feature distributions
  • Concept drift: Model performance degradation
  • Staleness: Feature freshness, update frequency
  • Latency: P50, P95, P99 percentiles

ML System Design Template

1

Clarifying Questions (5 min)

  • Business objective & success metrics
  • Scale: users, QPS, data size
  • Latency requirements
  • Available data & labels
2

High-Level Design (5 min)

  • System diagram
  • ML vs non-ML components
  • Online vs offline processing
3

Data Pipeline (10 min)

  • Training data construction
  • Label generation
  • Feature engineering
  • Negative sampling strategy
4

Model Architecture (10 min)

  • Model choice & justification
  • Loss function
  • Training strategy
5

Evaluation (5 min)

  • Offline metrics
  • Online metrics
  • A/B testing plan
6

Serving & Production (5 min)

  • Inference pipeline
  • Cold start handling
  • Monitoring & retraining
